What is a Monad?

The quest for defining what a Monad is has created a large amount of terrible blog posts, metaphors and misleading comparisons. This website is meant to simply provide a correct definition without trying to make it more accessible. I will not explain what a monad is but will define its elements and laws.

Semigroup

In abstract algebra, a Semigroup is a non-empty set with an associative binary operation.

Type definition

class Semigroup a where (<>) :: a -> a -> a mappend :: a -> a -> a

Monoid

A Monoid is a Semigroup with a unit element. Monoid must have the same binary operation from Semigroup and adds a unit function mempty.

For every object m in category M, binary operation p and unit element u the equation p(u, m) = m = p(m, u) must hold true for every Monoid. In practical terms, a set of natural numbers N with a binary operation addition (+) and a unit element 0 is a Monoid.

Type definition

class Semigroup m => Monoid m where (<>) :: a -> a -> a mappend :: a -> a -> a mempty :: m

Laws

mappend must be associative
mempty is it's identity

Functor

Functors are structure preserving morphisms between categories. A functor F: A -> B maps each object a from category A to an object Fa of category B.
